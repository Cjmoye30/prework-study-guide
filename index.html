HTML

  • HTML - composed of elements which tell the page how to be displayed. And tags - which define the elements.
  • HTML code should be structured and organzied to firstly give the webpage a good foundation before proceeding to anything additional.
  • The head element contains information about the webpage. This metadata helps the browser render the page correctly. This is a semantic element
  • The body element contains the contents of the page. This is a semantic element.
  • Elements nested inside other elements are called child elements.
  • HTML attributes define additional characteristics about the elements which are alwasy defined in the opening tag.

Git & Github Section

Workflow and associated Commands

  • git status - shows the current branch you're located in.
  • git checkout -b feature/[name of the branch you're wanting to create] - Creates a new branch. Necessary to create a branch off of your updated main branch. Common standard to always call new branches features.
  • git commit -A - adds all modifications made to the staging area.
  • git commit -m "[commit message]" - Adds a commit message which must be written inside quotation marks.
  • git pull origin main - pulls in the most recent main branch. Important to always do this when others are working on the same project to have the most recent code. If not, issues can occur.
  • git push origin feature/[name of the ] - pushes your feature up to the main branch.
  • Once the code has been pushed successfully, it shows in your Github dashboard as a pull request which must be reviewed (preferably by your colleagues) before it can be merged into the main branch.
